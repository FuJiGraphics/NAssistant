# NAssistant Agent Notes

이 파일은 이후 Codex 세션이 프로젝트 의도를 잊지 않도록 남겨둔 작업 메모입니다.

## 프로젝트 정체성

NAssistant는 VS Code 확장입니다. 목표는 Claude, Codex, ChatGPT, Copilot Chat 등 특정 AI를 직접 다루는 것이 아니라, 어떤 AI를 쓰든 공통으로 필요한 "코드와 문맥을 잘 복사하고 정리하는 일"을 강력하게 보조하는 것입니다.

## 반드시 지킬 방향

- AI 비의존적으로 설계한다.
- 자체 AI 채팅 UI, 말풍선 UI, 모델 호출 기능을 중심에 두지 않는다.
- 작은 편의 기능을 심플하고 완성도 높게 제공한다.
- VS Code의 자연스러운 흐름인 명령 팔레트, 단축키, 컨텍스트 메뉴, 클립보드를 우선한다.
- 사용자가 의도한 코드 범위, 파일 경로, 라인 번호, 언어 정보를 정확히 보존한다.
- 전체 파일 과복사, 불필요한 컨텍스트 포함, 민감 정보 포함 가능성을 줄이는 UX를 선호한다.

## 기술 스택

- VS Code extension
- TypeScript
- npm
- 명령은 실제 사용자 기능이 있을 때만 추가한다.
- Activity Bar에는 NAssistant 진입점이 있을 수 있지만, 실제 기능 없이 무거운 UI를 만들지 않는다.
- 기본 `Ctrl+C`/`Command+C`는 오버라이드하지 않는다.
- AI용 위치 복사는 명시적 명령/단축키인 `Ctrl+Shift+C`/`Command+Shift+C`로만 제공한다.
- AI용 위치 복사는 코드 본문을 복사하지 않고 `[location: path:start-end]`만 클립보드에 넣는다.
- Explorer 파일 참조 복사도 `Ctrl+Shift+C`/`Command+Shift+C`를 사용하되 파일 본문이나 첨부가 아니라 `[file: path]` 참조만 복사한다.
- 붙여넣기 대상이 챗봇인지 일반 에디터인지 안정적으로 판별할 수 있다고 가정하지 않는다.

## 좋은 기능의 기준

좋은 기능은 "AI가 무엇인지 몰라도 유용한가?"에 예라고 답할 수 있어야 합니다.

예:

- 선택 영역을 AI 컨텍스트 형식으로 복사
- 현재 파일 또는 선택 영역의 파일 참조 복사
- 열린 에디터들의 선택 영역만 묶어 복사
- 짧은 프롬프트 템플릿으로 복사 내용 감싸기
- 복사된 파일 수와 라인 수를 피드백으로 보여주기

## UI 일원화 메모

- Webview 안의 선택 UI는 네이티브 `<select>`보다 프로젝트 스타일의 커스텀 드롭다운을 우선한다.
- 드롭다운은 트리/옵션 패널의 실제 버튼 폭에 맞춰 아래로 붙어서 열리는 느낌을 준다.
- 메뉴가 위로 뜨거나, 버튼과 메뉴 폭이 어긋나거나, 별도 OS 팝업처럼 보이는 UX는 피한다.
- 팝업/드롭다운은 VS Code 테마 변수 기반으로 심플하게 만들고, 과한 장식보다 규격, 간격, 선택 상태의 명확성을 우선한다.

## 피해야 할 방향

- Cursor류의 통합 AI IDE를 따라 만들기
- 특정 AI 서비스에 종속되는 API 래퍼 만들기
- 많은 기능을 넓게 추가해서 핵심 사용성을 흐리기
- 설명이 많은 화면이나 무거운 설정 UI부터 만들기

## 커밋 규칙

- 이 프로젝트의 커밋 작성자 계정은 항상 `fujigraphics`를 사용한다.
- 커밋 타입은 `feat`, `chore`, `fix`, `refactor`를 사용한다.
- `feat`: 기능 설명
- `chore`: 코드 수정 없는 작업
- `fix`: 버그 수정
- `refactor`: 기존 구조에서 개선하거나 변경
- 커밋 설명은 간결하게 1~5줄의 나열 방식으로 작성한다.
- 커밋 메시지에는 커밋이나 문서를 작성한 주체로 AI를 언급하지 않는다.

## 제품 문장

NAssistant는 AI를 조종하는 도구가 아니라, AI에게 건네는 코드 문맥을 정확하게 준비하는 도구다.
